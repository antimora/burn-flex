# burn-flex-bench-candle

Direct per-op comparison between `burn-flex` and `candle-core` on CPU, using
pure-Rust kernels on both sides (no BLAS, no Accelerate, no MKL).

The goal is to attribute the end-to-end gap observed in
[this wav2vec2-large benchmark](https://gist.github.com/dnvt/468b2bb464c926c58ad7da5571e63abe)
to specific ops so we know where to invest.

## Why a separate crate

`burn-flex` is planned to be merged upstream into `burn`, so nothing that
pulls in `candle-core` belongs in its dependency graph. This crate is
`publish = false` and exists only for benchmarking.

## Running

```sh
cargo bench -p burn-flex-bench-candle --bench matmul
cargo bench -p burn-flex-bench-candle --bench conv1d
```

Both backends receive byte-identical input data generated by the same
deterministic filler.

## Baseline: 2026-04-05, Apple M3 Max

### matmul (f32, square)

| N    | burn-flex | candle    | Winner       |
|------|-----------|-----------|--------------|
| 128  | 43.6 us   | 119.1 us  | flex 2.73x   |
| 256  | 166.3 us  | 160.4 us  | ~tie         |
| 512  | 581.8 us  | 587.3 us  | ~tie         |
| 1024 | 2.818 ms  | 2.634 ms  | candle 1.07x |

Matmul is essentially a dead heat on square shapes. Flex wins at 128 (likely
gemm dispatch overhead in candle), candle edges ahead at 1024. The gist's
wav2vec2 gap is *not* coming from matmul on this hardware.

### conv1d (wav2vec2-large feature extractor, 1s of 16kHz audio)

Seven strided conv1d layers that downsample raw audio 320x.

#### Baseline (2026-04-05, before optimization)

| Layer | Shape                    | burn-flex | candle    | Winner         |
|-------|--------------------------|-----------|-----------|----------------|
| L0    | 1x1x16000,   k=10, s=5   | 517 us    | 3.631 ms  | **flex 7.0x**  |
| L1    | 1x512x3199,  k=3,  s=2   | 8.104 ms  | 6.720 ms  | candle 1.21x   |
| L2    | 1x512x1599,  k=3,  s=2   | 4.293 ms  | 3.683 ms  | candle 1.17x   |
| L3    | 1x512x799,   k=3,  s=2   | 3.473 ms  | 2.045 ms  | candle 1.70x   |
| L4    | 1x512x399,   k=3,  s=2   | 2.732 ms  | 1.254 ms  | candle 2.18x   |
| L5    | 1x512x199,   k=2,  s=2   | 1.674 ms  | 660 us    | candle 2.54x   |
| L6    | 1x512x99,    k=2,  s=2   | 1.453 ms  | 468 us    | candle 3.10x   |
| **Total** |                      | **22.25 ms** | **18.46 ms** | candle 1.21x |

#### After weight-transpose fix (2026-04-05)

A single ~15-line change in `conv3d_impl` rewrote the `(c_in, K) -> (K, c_in)`
weight transpose as a tight `K`-inner loop that LLVM autovectorizes. This was
the dominant cost for small-shape conv.

| Layer | burn-flex | candle    | Winner         | vs baseline  |
|-------|-----------|-----------|----------------|--------------|
| L0    | 498 us    | 4.05 ms   | **flex 8.1x**  | ~unchanged   |
| L1    | 6.75 ms   | 6.64 ms   | tie            | **-17%**     |
| L2    | 3.29 ms   | 3.60 ms   | **flex 1.09x** | **-23%**     |
| L3    | 2.38 ms   | 1.99 ms   | candle 1.20x   | **-31%**     |
| L4    | 1.67 ms   | 1.22 ms   | candle 1.37x   | **-39%**     |
| L5    | 956 us    | 666 us    | candle 1.44x   | **-43%**     |
| L6    | 768 us    | 479 us    | candle 1.60x   | **-47%**     |
| **Total** | **16.27 ms** | **18.20 ms** | **flex 1.12x** | **-27%** |

#### After direct conv path (2026-04-06, current)

For L2-L6, decompose the convolution into `kw` gemm calls operating directly
on NCHW data with strided pointers. This eliminates both the NHWC conversion
and im2col buffer entirely. Fixed in
[antimora/burn-flex#34](https://github.com/antimora/burn-flex/issues/34).

| Layer | burn-flex | candle    | Winner         | vs baseline  |
|-------|-----------|-----------|----------------|--------------|
| L0    | 505 us    | 3.93 ms   | **flex 7.8x**  | ~unchanged   |
| L1    | 6.65 ms   | 6.48 ms   | tie            | **-18%**     |
| L2    | 2.40 ms   | 3.55 ms   | **flex 1.48x** | **-44%**     |
| L3    | 1.68 ms   | 2.02 ms   | **flex 1.20x** | **-52%**     |
| L4    | 1.31 ms   | 1.30 ms   | tie            | **-52%**     |
| L5    | 447 us    | 722 us    | **flex 1.62x** | **-73%**     |
| L6    | 306 us    | 530 us    | **flex 1.73x** | **-79%**     |
| **Total** | **13.34 ms** | **18.54 ms** | **flex 1.39x** | **-40%** |

burn-flex now **beats candle on every conv1d layer** except L1 (tied) and L4
(tied). The biggest wins are on the smallest layers where im2col overhead was
most significant relative to compute.
